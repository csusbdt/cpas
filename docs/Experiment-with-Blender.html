Preparation

Install Blender.

Read Advanced > Command Line > Introduction to learn how to run Blender from the command line. On my Mac, I launched Blender from the command line with the following command.

/Applications/blender.app/Contents/MacOS/blender

If you are running under Mac OS X, then you can create an alias to allow easier access to blender from the command line. One way to do this is add the following to .bash_profile in your home directory.

alias blender=/Applications/blender.app/Contents/MacOS/blender

Note that the above command I used is different than the example shown in the Blender documentation. You may need to similarly adjust what you read in the documentation.

Take careful notice of Argument Order when running Blender from the command line.

Experiment with the command line

When you first start Blender, it will show a cube. This cube is in a scene with a camera pointed at it. When you render the scene, you do it through this camera. In this experiment, we will generate an image of the cube as seen through the camera.

Run Blender and observe the grey box.

Save the current state of Blender as a Blend-file. Save it into a convenient location and name the file cube.blend.

From the command line, run the following to produce the first frame of animation (frame 0) as seen through the camera.

blender -b cube.blend -o //#.png -f 0

This command will generate an image file named 0.png. Verify that this file is an image of the Blender scene you saved. The -b in the command specifies that Blender is to run in the background. In other words, the command will not open the Blender application Window.

To better understand Blender's command line operation, read the following.

  • Command Line
  • Docs » Render » Workflows » Command Line

The above command uses the blender render engine; try the following to see the results with the Cycles render engine.

blender -b box.blend -o //#.png --engine CYCLES -f 0

Let's now run a trivial Python script through Blender on the command line. For this purpose, create hi.py with a single line of Python code: print('hi'). Test that the script runs as expected.

python hi.py

The script should display hi.

Now have Blender run the script.

blender -b -P hi.py

This command produces the same result: it displays hi (along with some extra information from Blender).

To prepare yourself for eventual problems, repeat the above experiment with a syntax error. For example, you could change hi.py to xprint('hi').

Read and do the Docs » Editors » Python Console tutorial.

In the Blender editor, SHIFT-F4 selects Console view and SHIFT-F5 selects 3D view.

The next thing I read was Data System.

The following are notes I took from Data System > Introduction.

  • The Blender UI includes an Outliner editor that lets you view the hierarchical structure of the blend-file.
  • Various data such as images can be packed in a blend-file.
  • Most data in a blend-file is in a data-block although some data is not, such as bones, sequence strips and vertex groups.
  • Use custom properties to store arbitrary data in data-blocks, bones, sequence strips.

The following are notes I took from Data System > Scenes

The scene properties that may be relevant to our problem include the following.

  • camera -- currently selected camera

The following are notes I took from Data System > Relative Paths.

You cannot enter relative paths into a new untitled blend-file; save it before linking to external files.

From the Save As function, you have the option to select Remap Relative to relocate a blend-file relative to its linked resources.
