Raindrops

Write a program that simulates raindrops hitting a flat surface.

Represent the appearance of a raindrop by drawing a small dot on the screen. Start by having the raindrops fall at some rate such as 2 drops per second. This means your program needs to draw a new dot every .5 seconds.

You need to determine a random position on the screen to display the dot. For this purpose, you can use the uniform function of the random package to generate random values that are restricted to an interval. The following is an example of how to use this function to generate a random x value between 0 and 300.

x = random.uniform(0, 300)

After you get the basic animation working, extend your code so that each dot lives for a certain amount of time. After a fallen raindrop has lived for this amount of time, stop drawing its dot.

In nature, the time between successive raindrops will be random. If we assume that the average number raindrops appearing in a unit of time is constant and the appearance of raindrops are independent of each other, then the time between successive raindrops can be modeled by an exponential distribution. If we let lambda represent the average number of raindrops that appear in a unit of time, then random.expovariate(1/lambda) will produce times to the next raindrop that appear random and occur lambda times per second on average. The following code shows how to generate such a value.

import random

lambda = 1.0  # average number of raindrops per second

# Determine the time to next raindrop.
t = random.expovariate(1/lambda)

Experiment with different values for lambda to see the effects. Does this process improve the animation in the sense of making it more realistic?

Try to make your animation even more convincing by making the evaporation times normally distributed. You can use the normalvariate function of the random package to generate values that follow a normal distribution.

random.normalvariate(mean, std_dev)

Does this added work produce an animation that is more convincing?
